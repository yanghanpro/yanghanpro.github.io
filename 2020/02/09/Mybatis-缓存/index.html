Mybatis 缓存

无标签
发布日期:   2020-02-09

if：判断

要求：动态查询员工信息，携带了任何信息都可以查询员工

寻常方法：

<select id="getEmpByConditionIf" resultType="com.yang.bean.Employee">
    select * from tbl_employee
    where
    <!-- test属性中是OGNL表达式，类似EL表达式 -->
    <if test="id!=null">
        and id = #{id}
    </if>
    <!-- 如果包含特殊符号，需要转义 -->
    <if test="lastName!=null and lastName!=&quot;&quot;">
        and last_name like #{lastName}
    </if>
    <!-- OGNL会自动进行字符串和数字转换 -->
    <if test="gender==0 or gender==1">
        and gender = #{gender}
    </if>
    <if test="email!=null and email.trim()!=&quot;&quot;">
        and email = #{email}
    </if>
</select>

where：字符串截取 | 封装查询条件

但是这样会有问题，当id为null时，就会多出一个and，报错

### SQL: select * from tbl_employee where and last_name like ?

为了解决and问题，有2种方法：

  • 【普通】where 1=1

  • 【推荐】使用where标签，将所有查询条件包含在内，其中所有SQL语句之前都加and / or，mybatis会将where标签内的sql，多出来的一个and或or去掉

<select id="getEmpByConditionIf" resultType="com.yang.bean.Employee">
    select * from tbl_employee
    where 1=1
    <if test="id!=null">
        and id = #{id}
    </if>
    <if test="lastName!=null and lastName!=&quot;&quot;">
        and last_name like #{lastName}
    </if>
    <if test="gender==0 or gender==1">
        and gender = #{gender}
    </if>
    <if test="email!=null and email.trim()!=&quot;&quot;">
        and email = #{email}
    </if>
</select>
<select id="getEmpByConditionIf" resultType="com.yang.bean.Employee">
    select * from tbl_employee
    <where>
        <if test="id!=null">
            and id = #{id}
        </if>
        <if test="lastName!=null and lastName!=&quot;&quot;">
            and last_name like #{lastName}
        </if>
        <if test="gender==0 or gender==1">
            and gender = #{gender}
        </if>
        <if test="email!=null and email.trim()!=&quot;&quot;">
            and email = #{email}
        </if>
    </where>
</select>

注意事项：where只能去掉第一个多出来的or或and，而且必须在前面

trim：字符串截取（万金油）

如果把and放在SQL语句后面，那么where标签无法去除。

因此我们可以使用trim标签解决，trim标签体中是整个字符串拼接后的结果

  • prefix：前缀，给拼串后的整个SQL字符串加一个前缀
  • prefixOverrides：前缀覆盖，去掉整个字符串前面多余的字符
  • suffix：后缀，给拼串后的整个SQL字符串加一个后缀
  • suffixOverrides：后缀覆盖，去掉整个字符串后面多余的字符
<!-- List<Employee> getEmpByConditionTrim(Employee employee); -->
<select id="getEmpByConditionTrim" resultType="com.yang.bean.Employee">
    select * from tbl_employee
    <trim prefix="where" suffixOverrides="and">
        <if test="id!=null">
            id = #{id} and
        </if>
        <if test="lastName!=null and lastName!=&quot;&quot;">
            last_name like #{lastName} and
        </if>
        <if test="gender==0 or gender==1">
            gender = #{gender} and
        </if>
        <if test="email!=null and email.trim()!=&quot;&quot;">
            email = #{email}
        </if>
    </trim>
</select>

choose：分支选择

我们还可以使用分支选择，类似Java中带有break的switch-case，只会进入其中一个分支。如果有多个查询属性，只会进入第一个。

<!-- List<Employee> getEmpByConditionChoose(Employee employee); -->
<select id="getEmpByConditionChoose" resultType="com.yang.bean.Employee">
    select * from tbl_employee
    <where>
        <choose>
            <when test="id!=null">
                id=#{id}
            </when>
            <when test="lastName!=null">
                last_name like #{lastName}
            </when>
            <when test="email!=null">
                email = #{email}
            </when>
            <otherwise>
                gender = 0
            </otherwise>
        </choose>
    </where>
</select>

Set：字符串截取 | 封装更新条件

当我们要修改某条数据时，并不是所有字段都更新，而是只更新被修改的字段，因此首先我们考虑使用if标签。

<!-- int updateEmpByConditionSet(Employee employee); -->
<update id="updateEmpByConditionSet">
    update tbl_employee
    set
    <if test="lastName!=null and lastName!=&quot;&quot;">
        last_name = #{lastName},
    </if>
    <if test="gender==0 or gender==1">
        gender = #{gender},
    </if>
    <if test="email!=null and email.trim()!=&quot;&quot;">
        email = #{email}
    </if>
    where id = #{id}
</update>

    }

可以看到报错了，原因是多了个逗号。

因此Mybatis提供了Set标签解决多余的逗号问题。

<!-- int updateEmpByConditionSet(Employee employee); -->
<update id="updateEmpByConditionSet">
    update tbl_employee
    <set>
        <if test="lastName!=null and lastName!=&quot;&quot;">
            last_name = #{lastName},
        </if>
        <if test="gender==0 or gender==1">
            gender = #{gender},
        </if>
        <if test="email!=null and email.trim()!=&quot;&quot;">
            email = #{email}
        </if>
    </set>
    where id = #{id}

</update>

这个问题也可以使用trim标签解决：加上前置set，去掉多余后缀“，”

（trim标签真香~）

<!-- int updateEmpByConditionTrim(Employee employee); -->
<update id="updateEmpByConditionTrim">
    update tbl_employee
    <trim prefix="set" suffixOverrides=",">
        <if test="lastName!=null and lastName!=&quot;&quot;">
            last_name = #{lastName},
        </if>
        <if test="gender==0 or gender==1">
            gender = #{gender},
        </if>
        <if test="email!=null and email.trim()!=&quot;&quot;">
            email = #{email}
        </if>
    </trim>
    where id = #{id}

</update>

内置参数

在这些OGNL表达式中，我们可以使用方法传递过来的参数进行判断和取值

Mybatis也提供了2个内置参数

  • _parameter：代表整个参数
    • 单参：_parameter就是该参数
    • 多参：参数被封装为一个map，_parameter就是这个map
  • _databaseId：在配置了databaseIdProvider标签后，_databaseId就代表当前数据库的别名
    <!-- List<Employee> getEmpByInnerParameter(Employee employee); -->
    <select id="getEmpByInnerParameter" resultType="com.yang.bean.Employee">
        <if test="_databaseId=='mysql'">
            select * from tbl_employee
            <if test="_parameter!=null">
                where id = #{_parameter.id} and last_name = #{lastName}
            </if>
        </if>
        <if test="_databaseId=='oracle'">
            select * from tbl_employee
            <if test="_parameter!=null">
                where id = #{_parameter.id} and last_name = #{lastName}
            </if>
        </if>
    </select>

bind：绑定

我们在做模糊查询时，需要写%或_

这些符号通常写在SQL语句中或测试中

  • 【推荐，灵活7】employee.setLaseName(“%大%”);——>#{lastName}

  • 【有安全问题】employee.setLaseName(“大”);——>’%${lastName}’

还有一种方法。

bind标签可以将OGNL表达式的值绑定到一个变量上，方便后来引用该变量

通过bind标签可以有效的将特殊符号提取出来，在需要的SQL语句中引入即可。

<!-- List<Employee> getEmpByLastNameWithBind(Employee employee); -->
<select id="getEmpByLastNameWithBind" resultType="com.yang.bean.Employee">
    <bind name="_lastName" value="'%'+lastName+'%'"/>
    select * from tbl_employee where last_name like #{_lastName}
</select>
