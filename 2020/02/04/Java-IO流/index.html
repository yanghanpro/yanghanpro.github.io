Java IO流

Java IO流
Java
发布日期:   2020-02-04

1. IO流 概述

1.1. 位置

java.io包下，

1.2. IO流 简述

IO == Input/Output ，用于处理设备之间的数据传输

输入input：读取外部数据（磁盘、光盘等存储设备的数据）到程序（内存）中。

输出output：将程序（内存）数据输出到磁盘、光盘等存储设备中

因此我们是站在内存角度上看输入输出的。

Java程序中，对于数据的输入/输出操作以“流(stream)” 的方式进行。

java.io包下提供了各种“流”类和接口，用以获取不同种类的数据，并通过标准的方法输入或输出数据

1.3. IO流 分类

Java的IO流共涉及40多个类，实际上非常规则，都是从如下4个抽象基类派生的。

由这四个类派生出来的子类名称都是以其父类名作为子类名后缀。

（抽象基类） 字节流 字符流
输入流 InputStream Reader
输出流 OutputStream Writer

2. 节点流（文件流）

FileReader（磁盘 读入到 内存）
int read() 返回读入的一个字符 如果读到末尾了，返回-1
int read(char [] c) 返回读入的多个字符组成字符数组的长度 如果读到末尾了，返回-1
int read(char [] c, int off, int len)

注意事项：

异常处理：程序中打开的文件 IO 资源不属于内存里的资源，垃圾回收机制无法回收该资源，所以应该显式关闭文件 IO 资源。为保证流资源一定可以执行关闭操作，需要使用try-catch-finally处理。

读入的文件一定要保证存在，否则会报异常：FileNotFoundException

// int read()
@Test
public void FileReaderTest01() throws Exception {
    FileReader fr = null;
    try{
        // 1.创建File对象
        File file = new File("demo.txt");

        // 2.创建流对象
        fr = new FileReader(file);

        // 3.读入数据
        // 返回读入的一个字符（char转为int），若达到文件末尾，返回-1
        // int data = fr.read();
        // while (data != -1){
        //         System.out.print((char)data);
        //         // 游标向下，读入下一个字符
        //        data = fr.read();
        // }
        // 3.读入数据（语法优化）
        int data;
        while ((data = fr.read()) != -1){
            System.out.print((char)data);
        }

    }catch(Exception e){
        e.printStackTrace();
    }finally{
        // 4.关闭流（一定要关闭，否则GC无法处理）
        try{
            fr.close();
        }catch(Exception e){
            e.printStackTrace();
        }
    }
}

// int read(char [])
@Test
    public void FileReaderTest02() throws Exception {
        FileReader fr = null;
        try{
            // 1.创建File对象
            File file = new File("demo.txt");

            // 2.创建流对象
            fr = new FileReader(file);

            // 3.读入数据
            char[] cbuf = new char[5];
            int len;
            while ((len = fr.read(cbuf)) != -1){
                /*
                 *  错误写法：cbuf的长度是固定的，
                 *  但是最后一次获取的长度不一定是装满char[]，因此会出现多余
                 *  for (int i=0;i<cbuf.length;i++){}
                 */
                // 方式一：正确写法
                for (int i=0;i<len;i++){
                    System.out.print(cbuf[i]);
                }
                /*
                    String s = new String(cbuf);
                    错误写法：原因与上面的一致
                 */
                // 方式二：正确写法
                String s = new String(cbuf,0,len);
                System.out.println(s);
            }
        }catch(Exception e){
            e.printStackTrace();
        }finally{
            // 4.关闭流（一定要关闭，否则GC无法处理）
            try{
                fr.close();
            }catch(Exception e){
                e.printStackTrace();
            }
        }
    }

FileWriter（内存 写出到 磁盘）

注意事项：

写出的目标文件可以不必存在。

当不存在时，在输出过程中会自动创建文件。

如果存在，

使用构造器FileWriter(file,false) 或 FileWrter(file)时，写出是覆盖目标文件的原有内容。

使用构造器FileWriter(file,false) 时，是在原有文件内容上追加。

@Test
    public void FWDemo02() {
        FileWriter fw = null;
        try {
            // 1.创建File类对象
            File file = new File("demo.txt");

            // 2.创建字符输出流对象
            fw = new FileWriter(file);

            // 3.执行写出操作
            fw.write("\nI have a Dream!");
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // 4.关闭流
            if (fw != null){
                try {
                    fw.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }   
    }

    public void FWDemo02() throws IOException {

        FileWriter fw = null;

分割线

注意

字符流无法处理图片文件（图片是字节数据）

字节流无法很好处理文本文件（文本是字符数据，一个汉字三个字符，可能会出现乱码）

FileInputStream + FileOutputStream

复制图片

注意：FileInputStream 和 FileOutputStream可以复制文本文件！

@Test
    public void FileStreamdemo01(){
        FileInputStream fis = null;
        FileOutputStream fos = null;
        try {
            File srcFile = new File("xxx.jpg");
            File destFile = new File("xxx-1.jpg");

            fis = new FileInputStream(srcFile);
            fos = new FileOutputStream(destFile);

            // 复制过程
            byte[] buffer = new byte[5];
            int len;
            while ((len = fis.read(buffer)) != -1){
                fos.write(buffer,0,len);
            }
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // 4.关闭流对象
            try {
                if (fis != null){
                    fis.close();
                }
            } catch (IOException e) {
                e.printStackTrace();
            }
            try {
                if (fos != null){
                    fos.close();
                }
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }

3.缓冲流（处理流的一种）

作用：提高文件的读写效率

@Test
public void BufferDemo01() throws FileNotFoundException {
    BufferedInputStream bis = null;
    BufferedOutputStream bos = null;
    try {
        // 1.造文件
        File srcFile = new File("xxx.jpg");
        File destFile = new File("xxx-1.jpg");

        // 2. 造流
        // 2.1 造节点流
        FileInputStream fis = new FileInputStream(srcFile);
        FileOutputStream fos = new FileOutputStream(destFile);
        //2.2 造缓冲流
        bis = new BufferedInputStream(fis);
        bos = new BufferedOutputStream(fos);

        // 3.复制（读取 + 写入）
        byte[] buffer = new byte[10];
        int len;
        while ((len = bis.read(buffer)) != -1){
            bos.write(buffer,0,len);
        }
    } catch (IOException e) {
        e.printStackTrace();
    } finally {
        // 4.资源关闭
        // 注意：先关闭外层流，再关闭内层流
        // 关闭外层流的同时，也会关闭内层流，因此可以省略。
        if (bis !=null){
            try {
                bis.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
        if (bos !=null){
            try {
                bos.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
