映射文件 结构

增删改查 标签

注意事项：

  • 接口中的增删改方法：可以定义这些类型的返回值：void / Integer /Long / Boolean
  • 注意增删改需要提交数据
    • 手动提交：sqlSessionFactory.openSession(); sqlSession.commit();
    • 自动提交：sqlSessionFactory.openSession(true);

获取自增主键的值

我们先来做一个插入测试：

<-- int addEmp(Employee employee); -->
<insert id="addEmp">
    insert into tbl_employee(last_name, gender, email)
    values (#{lastName}, #{gender}, #{email})
</insert>
@Test
public void addTest() throws IOException {
    SqlSessionFactory sqlSessionFactory = getSqlSessionFactory();
    SqlSession sqlSession = sqlSessionFactory.openSession(true);
    try{
        EmployeeMapper mapper = sqlSession.getMapper(EmployeeMapper.class);
        Employee employee = new Employee(null, "大白", "1", "dabai@qq.com");
        int count = mapper.addEmp(employee);
        if (count > 0){
            System.out.println("插入成功:" + employee);
        }else {
            System.out.println("插入失败！");
        }
    }finally {
        sqlSession.close();
    }
}
DEBUG 02-08 11:06:34,380 ==>  Preparing: insert into tbl_employee(last_name, gender, email) values (?, ?, ?)   (BaseJdbcLogger.java:159) 
DEBUG 02-08 11:06:34,416 ==> Parameters: 大白(String), 1(String), dabai@qq.com(String)  (BaseJdbcLogger.java:159) 
DEBUG 02-08 11:06:34,446 <==    Updates: 1  (BaseJdbcLogger.java:159) 
插入成功:employee{id=null, lastName='大白', gender='1', email='dabai@qq.com'}

此时我们看到插入成功了，但是数据少了一个id

这是因为【Mysql】支持【主键自增】，我们在执行插入操作时不用指定自增主键的值。

那么我们如果需要这个自增主键的值，要怎么做呢？

回忆JDBC，是使用statement.getGeneratedKeys()方法获取主键的。

同样的，在Mybatis中，也是使用该方法获取主键值的。

  • useGeneratedKeys=”true” ：使用自增主键获取主键值策略

  • keyProperty=”id”：指定主键对应的JavaBean属性（就是把主键值封装到该javaBean的这个属性中）

<insert id="addEmp" useGeneratedKeys="true" keyProperty="id">
    insert into tbl_employee(last_name, gender, email)
    values (#{lastName}, #{gender}, #{email})
</insert>
DEBUG 02-08 11:13:44,496 ==>  Preparing: insert into tbl_employee(last_name, gender, email) values (?, ?, ?)   (BaseJdbcLogger.java:159) 
DEBUG 02-08 11:13:44,532 ==> Parameters: 大白(String), 1(String), dabai@qq.com(String)  (BaseJdbcLogger.java:159) 
DEBUG 02-08 11:13:44,539 <==    Updates: 1  (BaseJdbcLogger.java:159) 
插入成功:employee{id=3, lastName='大白', gender='1', email='dabai@qq.com'}

接口中的参数处理

单个参数：使用 #{参数名}取出参数值

多个参数：

方法：Employee getEmpByIdAndName(Integer id, String lastName);
取值：#{id},#{lastName}
测试：Employee emp = mapper.getEmpByIdAndName(1, "小鞠");
报错：
Cause: org.apache.ibatis.binding.BindingException: Parameter 'id' not found. Available parameters are [arg1, arg0, param1, param2]

Mybatis底层做了特殊处理，多个参数会被封装成一个map，#{}就是从map中获取指定key的值

key：param1,param2, … ，或者参数的索引也可以。

value：传入的参数值

方法：Employee getEmpByIdAndName(Integer id, String lastName);
取值：#{param1}, #{param2}
测试：Employee emp = mapper.getEmpByIdAndName(1, "小鞠");
输出：employee{id=1, lastName='小鞠', gender='0', email='xiaoju@qq.com'}

但是这样既混乱又不美观，我们可以在接口的参数前使用@Param("id")注解

命名参数：给参数命名，@Param("id")明确指定封装参数时map的key，多个参数会被封装成一个map

key：使用@Param("id")注解指定的值

value：参数值

方法：Employee getEmpByIdAndName(@Param("id") Integer id, @Param("lastName") String lastName);
取值：#{id},#{lastName}
测试：Employee emp = mapper.getEmpByIdAndName(1, "小鞠");
输出：employee{id=1, lastName='小鞠', gender='0', email='xiaoju@qq.com'}

JavaBean对象：

若多个参数正好是业务的数据模型，可以直接传入JavaBean对象

使用#{属性名}取出传入的JavaBean对象的属性值

Map:

若多个参数不是业务模型中的数据，不经常使用，我们可以传入map

使用#{key}取出map中对应的值

方法：Employee getEmpBymap(Map<String,Object> map);
取值：#{id},#{lastName}
测试：
    Map<String, Object> map = new HashMap<String, Object>();
    map.put("id","1");
    map.put("lastName","小鞠");
    Employee emp = mapper.getEmpBymap(map);
输出：employee{id=1, lastName='小鞠', gender='0', email='xiaoju@qq.com'}

List:

若参数是数组或集合类（Collection：List / Set），也会将其封装到map中

key：collection / list

value：collection[0] / list[0]，…

方法：Employee getEmpByIdList(List<Integer> ids);
取值：#{list[0]}
测试：
    List<Integer> list = new ArrayList();
    list.add(1);
    Employee emp = mapper.getEmpByIdList(list);
输出：employee{id=1, lastName='小鞠', gender='0', email='xiaoju@qq.com'}

区别 #与$取值

#{}和${}都可以获取map中的值 / JavaBean对象属性值

区别：

${}：以预编译形式，将参数设置到SQL语句中（PreparedStatement，防止SQL注入）

#{}：取出的值直接拼接在SQL语句中，会有安全问题

select标签的返回值类型（集合：List / Map）

之前的案例中，接口中的查询方法的返回值类型都是JavaBean类，

如果返回值是集合类，他的内部构造是什么样子？

List

封装JavaBean对象的List，其传参方式和正常JavaBean对象相同

Map

封装单条JavaBean对象的Map，其结构为：k（列名）——v（列值）

封装多条JavaBean对象的Map，其结构为：k（主键）—— v（JavaBean对象）

public SqlSessionFactory getSqlSessionFactory() throws IOException {
    String resource = "mybatis-config.xml";
    InputStream inputStream = Resources.getResourceAsStream(resource);
    SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
    return sqlSessionFactory;
}

@Test
public void addTest() throws IOException {
    SqlSessionFactory sqlSessionFactory = getSqlSessionFactory();
    SqlSession sqlSession = sqlSessionFactory.openSession(true);
    try{
        EmployeeMapper mapper = sqlSession.getMapper(EmployeeMapper.class);


        List<Employee> empList = mapper.getEmpLikeName("%小%");
        for (Employee emp : empList){
            System.out.println(emp);
        }
        //employee{id=1, lastName='小鞠', gender='0', email='xiaoju@qq.com'}
        //employee{id=5, lastName='小明', gender='1', email='xiaoming@qq.com'}


        Map<String, Object> empMap = mapper.getEmpByMapId(1);
        System.out.println(empMap); 
        //{gender=0, last_name=小鞠, id=1, email=xiaoju@qq.com}


        Map<Integer, Employee> empsMap = mapper.getEmpLikeNameReturnMap("%小%");
        System.out.println(empsMap);
        // {1={gender=0, last_name=小鞠, id=1, email=xiaoju@qq.com}, 5={gender=1, last_name=小明, id=5, email=xiaoming@qq.com}}
    }finally {
        sqlSession.close();
    }
}

resultMap：结果集映射规则

在之前，对于数据库字段（last_name）和属性名（lastName）不一致的问题

我们提供了2种方案：

  • 在【全局配置文件】中使用了【驼峰命名规则】
  • 在【XML映射文件】中的SQL语句给该字段起别名，别名为JavaBean对象的属性名

现在，有我们也可以在【XML映射文件】中使用【resultMap标签】来自定义结果集映射规则

只有<select>标签才有resultType和resultMap属性，<insert>、<delete>、<update>标签都没有

resultType和resultMap只能存在一个

解决字段不一致问题

<mapper namespace="com.yang.dao.EmployeeMapper">

    <resultMap id="MyEmp" type="com.yang.bean.Employee">
        <!-- 主键 -->
        <id property="id" column="id"/>
        <!-- 其余列 -->
        <result property="lastName" column="last_name"/>
        <result property="gender" column="gender"/>
        <result property="email" column="email"/>
    </resultMap>

    <select id="getEmpById" resultMap="MyEmp">
        select * from tbl_employee where id = #{id}
      </select>
</mapper>

    return sqlSessionFactory;

关联查询（一对一 | Bean对象）

之前的查询中，我们使用了resultType进行单表查询，如果进行关联查询的话，resultType则有些力不从心了。因为resultType无法明确结果集映射规则，导致dept属性为null。

这时候我们可以使用resultMap自定义结果集的映射规则

【嵌套联合查询】使用级联属性封装结果集

【嵌套联合查询】<association>定义单个对象的封装

<association>可以指定联合的JavaBean对象

  • property属性：指定哪个属性是联合的JavaBean对象
  • javaType：指定该属性对象的类型

【分步查询】<association>分步查询

【分步查询延迟加载】

在上面的分步拆查询案例中，执行了2次SQL，也就是访问了2次数据库，这样会导致过于频繁的访问数据库，如果我们只想要员工信息而不要部门信息，他还是会去查2次，造成资源浪费，因此Mybatis提供了懒加载策略。当我们需要的时候在加载，否则不加载。

关联查询（一对多 | 集合）

通过部门编号查询员工，员工信息封装为部门的一个集合属性

【嵌套联合查询】
package com.yang.bean;

import java.util.List;

public class Department {
    private Integer id;
    private String deptName;
    private List<Employee> emps;
    // getter setter toString
}
package com.yang.dao;
import com.yang.bean.Department;
public interface DepartmentMapper {
    Department getDeptById(Integer id);
}
<mapper namespace="com.yang.dao.DepartmentMapper">

    <resultMap id="myDept" type="com.yang.bean.Department">
        <id property="id" column="did"/>
        <result property="deptName" column="dept_name"/>
        <!-- 定义关联的员工集合属性的封装规则 -->
        <collection property="emps" ofType="com.yang.bean.Employee">
            <id property="id" column="eid"/>
            <result property="lastName" column="last_name"/>
            <result property="gender" column="gender"/>
            <result property="email" column="email"/>
        </collection>
    </resultMap>

    <!-- Department getDeptById(Integer id); -->
    <select id="getDeptById" resultMap="myDept">
        select d.id did, d.dept_name dept_name,
                e.id eid, e.last_name last_name, e.gender gender, e.email email
        from tbl_department d
        left join tbl_employee e
        on d.id = e.d_id
        where d.id = #{id}
    </select>
</mapper>
public class MybatisTest {

延迟加载

在collection标签中加入属性fetchType="lazy"表示延迟加载，fetchType="eager"表示立即加载

关于分步查询的column属性

不管是使用association查询单个对象，还是使用collection查询多个对象（集合），其中都要传入参数，而参数都依靠column属性传递，column可以传递一个参数或多个参数（将参数封装为map）

单参数：column=”id”

多参数：column=“{key1=column1, key2=column2}”，使用#{key1}接收参数
