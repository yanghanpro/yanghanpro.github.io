设计模式 代理模式

无标签
发布日期:   2020-02-03

静态代理

1. 静态代理的优劣

优点：

可以使真实角色（被代理角色）的操作更加纯粹，不用关注一些公共的业务

将公共业务交给代理角色，实现业务分工

公共业务发生扩展时，方便集中管理

缺点：

一个真实角色（被代理角色）就会产生一个代理角色，代码量翻倍，开发效率变低

2. 实例

(1) 接口

public interface Rent {
    void rent();
}

(2) 被代理角色

public class Host implements Rent{

    public void rent() {
        System.out.println("房东正在出租房屋...");
    }
}

(3) 代理角色

public class HouseProxy implements Rent{

    private Host host;

    public void setHost(Host host) {
        this.host = host;
    }

    public HouseProxy(Host host) {
        this.host = host;
    }

    public HouseProxy() {
    }

    // 代替“被代理角色”做事
    public void rent() {
        host.rent();
    }
}

(4) 测试（客户）

public class Client {
    public static void main(String[] args) {
        // “被代理角色”
        Host host =  new Host();
        // "代理角色" 与 “被代理角色” 接触
        HouseProxy houseProxy = new HouseProxy();
        houseProxy.setHost(host);
        // 用户与“代理角色”接触，并完成事务
        houseProxy.rent();
    }
}

静态代理（加深理解）

1. 导读

当一个Service中的每个方法都需要加上业务细节处理（如计算运行时间）时，如果将这些代码直接添加在每个方法中，无疑会降低代码的可读性，甚至还会导致代码出现问题。

因此，我们需要在不修改代码的基础上，使用代理来添加这些和业务不相干的代码。

2. 实例

(1) 接口

public interface UserService {
    void add();
    void delete();
    void update();
    void query();
}

(2) 被代理角色

public class UserServiceImpl implements UserService {
    public void add() {
        System.out.println("增加了一个用户..");
    }

    public void delete() {
        System.out.println("删除了一个用户..");
    }

    public void update() {
        System.out.println("修改了一个用户..");
    }

    public void query() {
        System.out.println("查询了一个用户..");
    }
}

(3) 代理角色

public class UserServiceProxy implements UserService {

    private UserServiceImpl userService;

    public void setUserService(UserServiceImpl userService) {
        this.userService = userService;
    }

    public void add() {
        log("add");
        userService.add();
    }

    public void delete() {
        log("delete");
        userService.delete();
    }

    public void update() {
        log("update");
        userService.update();
    }

    public void query() {
        log("query");
        userService.query();
    }

    // 日志方法
    public void log(String msg){
        System.out.println("[debug] 使用了" + msg + "方法");
    }
}

(4) 测试

public class Client {
    public static void main(String[] args) {
        UserServiceImpl userService = new UserServiceImpl();
        UserServiceProxy proxy = new UserServiceProxy();
        proxy.setUserService(userService);
        proxy.add();
        proxy.delete();
    }
}

动态代理

静态代理在不修改代码的情况下进行了纵向开发，但是有一个缺点：

一个 “被代理角色” 就会对应产生一个 “代理角色”

这无疑会让我们代码量成倍增多，使得开发效率变低，项目臃肿。

那么我们要思考：

有没有什么方法能够一次性的为所有的“被代理角色” 产生“代理角色” ？

答案是：有！那就是动态代理

1. 动态代理 概述

动态代理的底层：反射

与静态代理的一对一模式不同，动态代理的代理类是动态生成的，而不是写死的。

2. 动态代理的分类

基于接口的动态代理：JDK动态代理

基于类的动态代理：cglib，java字节码 JAVAssist

这边使用的是：JDK动态代理

3. 动态代理的好处

可以使真实角色（被代理角色）的操作更加纯粹，不用关注一些公共的业务

将公共业务交给代理角色，实现业务分工

公共业务发生扩展时，方便集中管理

一个动态代理类代理的是一个接口，这个接口可以修改

一个动态代理类可以代理实现同一接口的多个类

4. JDK动态代理

需要了解2个类：

java.lang.reflect.Proxy类（代理）

java.lang.reflect.InvocationHandler接口（调用处理程序）

invoke(Object proxy, method method, Object[] args)：处理被代理对象的方法并返回结果

invoke(被代理对象，被代理对象中的某个方法，方法参数)

5. 实例

(1) 代理类（很重要）

package com.yang.proxy.trendproxy;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

// 会自动生成代理类
public class ProxyInvocationHandler implements InvocationHandler {

    // 被代理的接口
    private Object target;

    public void setTarget(Object target) {
        this.target = target;
    }

    // 自动生成该接口的代理类
    public Object getProxy(){
        return Proxy.newProxyInstance(this.getClass().getClassLoader(),
                target.getClass().getInterfaces(),this);
    }

    // 会自动调用该方法：处理代理实例，并返回结果
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        // 通过反射处理
        log(method.getName());
        Object result = method.invoke(target, args);
        return result;
    }

    public void log(String msg){
        System.out.println("[DEBUG] 执行了" + msg +"方法...");
    }
}

(2) 测试

public class Client {
    public static void main(String[] args) {
        UserServiceImpl userService = new UserServiceImpl();

        ProxyInvocationHandler pih = new ProxyInvocationHandler();
        // 设置需要代理的接口
        pih.setTarget(userService);
        // 动态生成代理类
        UserService proxy =(UserService) pih.getProxy();
        // 自动调用invoke方法，扩展业务细节
        proxy.add();
        proxy.delete();
    }
}
