Java-集合框架(1)

无标签
发布日期:   2020-02-02

集合框架的概述

(1) 集合

集合和数组都是对多个数据进行存储操作（存入内存，即暂存）的结构，成为Java容器

(2) 集合与数组

集合 数组
长度 可变 固定
元素类型 可以多类型共存 不可变，类型单一，如String[ ] arr1, int[ ] arr2， Object arr3[ ]
增删改操作 ArrayList:查找快，增删慢
LinkedList:增删快，查找慢		 不方便，一旦数组满了，增操作就需要创建一个新数组
存储特点 List:有序，可重复
set:无序，不可重复		 有序，可重复

(3) 集合框架体系

Collection接口的常用方法

方法 描述
add (Object obj) 添加obj元素到集合中
size () 获取集合元素个数
addAll (Collection coll) 将集合coll添加到当前集合中
clear () 清空集合所有元素
isEmpty () 判断集合是否为空
contains (Object obj) 判断集合是否包含obj对象 底层是调用obj对象的equals( )
如果equals ()未重写返回false
如果equals ()已重写返回true
containsAll(Collection coll) 判断集合coll所有元素是否都存在于当前集合中
remove(Object obj) 移除当前集合中的obj元素 删除成功返回true
若无该元素返回false
removeAll(Collection coll) 移除当前集合中与coll包含的所有元素（移除交集，保留差集）
retainAll(Collection coll) 保留当前集合中与coll包含的所有元素（保留交集，移除差集）
equals(Collection coll) 判断当前集合和coll集合的元素是否相同 如果是List（有序），位置也需要相同
hashCode() 返回该集合的哈希值
toArray() 集合转换为数组 Arrays.asList()：数组转换为集合
Iterator iterator() 迭代器，遍历集合元素

遍历集合的方法

(1) Iterator接口

Iterator对象称为迭代器(设计模式的一种)，主要用于遍历 Collection 集合中的元素（无法遍历Map集合）

Collection接口继承了java.lang.Iterable接口，该接口有一个iterator()方法，那么所有实现了Collection接口的集合类都有一个iterator()方法，用以返回一个实现了Iterator接口的对象。

集合对象每次调用iterator()方法都得到一个全新的迭代器对象，默认游标都在集合的第一个元素之前。

方法 描述
boolean hasNext() 判断是否还有元素
Object next() 返回下一个元素
void remove() 在遍历集合时移除指定元素

注意：如果还未调用next()或在上一次调用 next 方法之后已经调用了 remove 方法，再调用remove都会报IllegalStateException

还未调用next()就调用remove()：如果游标在第一个，没有指向下一个，那remove的就是空的元素了）

上一次调用 next 方法之后已经调用了 remove 方法，再调用remove元素已经移除了，无法再次移除

(2) foreach / 增强for循环

Java 5.0 提供了 foreach 循环迭代访问 Collection和数组。

foreach可以用来遍历数组和集合。

遍历操作不需获取Collection或数组的长度，无需使用索引访问元素。

遍历集合的底层仍然时调用Iterator迭代器完成操作。

public void iteratorDemo(){
    Collection coll = new ArrayList();
        coll.add("AA");
        coll.add(123);  // 自动装箱
        coll.add(new Date());
        coll.add(false);
    // 遍历集合
    for(Object obj : coll){
        System.out.println(obj);
    }

    // 遍历数组
    int[] arr = new int[](1,2,3,4,5);
    for(int i: arr){
        System.out.println(i);
    }
}

(3) foreach面试题

public void Demo(){
    String[] arr = new String("MM","MM"."MM");

    // 使用普通for循环
    for(int i=0; i<arr.length;i++){
        arr[i]="GG";
    }
    for(int i=0;i<arr.length;i++){
        System.out.print(arr[i] + " ");    
    }

    /*------------------------------------*/

    // 使用增强for循环遍历
    for(String i : arr){
        i="GG";
    }
        for(int i=0;i<arr.length;i++){
        System.out.print(arr[i] + " ");    
    }
}
输出结果：
GG GG GG
MM MM MM
为什么？
因为普通for循环是直接对arr[i]进行操作，
而增强for循环是取出arr[i],将arr[i]赋值给String s,修改时只是修改了引用，但是没有修改数组元素的值。

List接口（Since 1.2）

特点：元素有序，可重复 ——-> ”动态数组“，本质是创建新数组，替换旧数组

(1) List接口的三个实现类

ArrayList：（Since 1.2）使用数组Obect[]存储，查询快，增删慢，线程不安全，效率高

LinkedList：（Since 1.2）使用双向链表存储，查询慢，增删快

Vector：（Since 1.0）使用数组Obect[]存储，查询快，增删慢，线程安全，效率低

三者异同点（面试题）

相同点：都实现了List接口，存储的数据都是有序可重复的。

不同点：见上

(2) List接口的常用方法

因为List有索引的概念（Set没有），因此除了Collection那些方法之外，List还有与索引相关的方法。

方法 描述
插入元素 void add(int index,Object ele) 在index位置插入ele元素
插入集合 boolean addAll(int index,Collection coll) 从index位置开始将coll集合中的所有元素添加进来
获取元素 Object get(int index) 获取指定index位置的元素
查找索引 int indexOf(Object obj) 返回元素obj在集合中首次出现的位置
查找索引 int lastIndexOf(Object obj) 返回元素obj在集合中末次出现的位置
移除元素 Object remove(int index) 移除index位置的元素，并返回index元素
覆盖元素 Object set(int index,Object obj) 设定index位置的元素为obj
截取集合 List subList(int fromIndex, int toIndex) 返回从fromIndex到toIndex位置的子集合

Arrays.asList(…) 方法返回的 List 集合，既不是 ArrayList 实例，也不是Vector 实例。 Arrays.asList(…) 返回值是一个固定长度的 List 集合

Set接口

无序，不可重复

无序性：是根据哈希值进行排序的（hashMap的key的排列规则），而不是按照存储顺序进行排序的。

不可重复性：计算新添加的对象的哈希值，若哈希值和已有元素的哈希值一样，再使用equals()进行比较，返回false就是没有重复。

Set接口是Collection的子接口，set接口没有提供额外的方法
Set 集合不允许包含相同的元素，如果试把两个相同的元素加入同一个Set 集合中，则添加操作失败。
Set 判断两个对象是否相同不是使用 == 运算符，而是根据 对象的equals() 方法

Set接口的三个实现类

HashSet：线程不安全，可以存储null值，底层是数组实现的。

​ LinkedHashSet：HashSet的子类，遍历数据时，可以按照添加的顺序遍历

TreeSet：可以按照添加对象的指定属性，进行排序
